- 文档版本：v0.1
- 产出日期：2026-03-27

## 2. 信息架构总览

平台首版信息架构分为六个一级模块：

1. 首页 / 岗位导航
2. 题库中心
3. 模拟面试
4. 笔试中心
5. LeetCode 练习
6. 候选人与内容治理后台

## 3. 页面结构

### 3.1 首页 / 岗位导航

- 岗位介绍：AI Infra 算力调度岗位职责、能力模型、考察维度。
- 训练入口：模拟面试、笔试、LeetCode 练习。
- 学习地图：基础知识、系统设计、调度策略、故障排查、编码训练。
- 个人进展：最近成绩、待补强模块、推荐题单。

### 3.2 题库中心

- 题库首页：按内容类型、专题、难度、岗位级别筛选。
- 题目详情：题干、考察点、作答要求、参考答案、评分标准、常见错误。
- 题单详情：专题题单、面试题单、笔试卷模板、训练路径。
- 标签管理：题目标签、能力标签、问题回流标签。

### 3.3 模拟面试

- 面试场次列表：待开始、进行中、已完成。
- 面试准备页：题单选择、面试官选择、候选人信息、目标岗位。
- 面试进行页：当前题目、计时、追问记录、实时备注、风险标记。
- 面试总结页：结构化面评、建议结论、问题回流入口。
- 面试记录页：历史记录、回放入口、评分对比。

### 3.4 笔试中心

- 笔试列表：按岗位模板和状态查看。
- 试卷详情：题目分布、总时长、评分方式。
- 作答页：题目导航、计时器、自动保存、交卷。
- 判卷页：客观题结果、主观题评分、错误标签。
- 笔试结果页：总分、分模块得分、改进建议。

### 3.5 LeetCode 练习

- 题目列表：专题、难度、状态筛选。
- 题目详情：问题描述、输入输出、样例、边界条件。
- 在线编码页：编辑器、运行、提交、结果反馈。
- 提交记录页：历史提交、耗时、错误原因、通过状态。

### 3.6 候选人与内容治理后台

- 候选人档案：基础信息、历史面试、笔试成绩、代码练习记录。
- 内容治理台：题目新增、修订、下线、标签维护。
- 问题回流池：按来源场次和问题类型查看待整理项。
- 数据看板：题目使用频次、区分度、通过率、错误分布。

## 4. 核心流程

### 4.1 候选人训练流程

1. 候选人进入首页选择训练目标。
2. 平台推荐题单或候选人主动筛选题库。
3. 候选人完成模拟面试、笔试或编码练习。
4. 系统沉淀训练记录并更新薄弱项画像。

### 4.2 面试官评审流程

1. 面试官创建面试场次并加载题单。
2. 面试过程中逐题记录表现与追问。
3. 面试结束后提交结构化面评。
4. 风险问题进入问题回流池，供题库治理使用。

### 4.3 题库治理流程

1. 题库维护者查看问题回流池。
2. 对问题进行分类：新增题目、修订题目、补充答案、优化标签。
3. 修订后同步到对应专题题单或笔试模板。
4. 周期性复盘题目效果与候选人表现数据。

## 5. 题库分类结构

### 5.1 一级分类

1. 面试题
2. 笔试题
3. LeetCode 练习

### 5.2 二级专题

围绕 AI Infra 算力调度岗位，建议首版覆盖以下专题：

1. Kubernetes 与容器基础
2. GPU / NPU 资源管理
3. 作业调度与队列策略
4. 多租户隔离与配额治理
5. 分布式系统与高可用
6. 网络、存储与数据通路
7. 监控告警与故障排查
8. 成本、容量与 SLA 管理
9. 大数据 / 训练任务编排
10. 编码与算法基础

### 5.3 标签维度

每道题建议至少挂载以下标签：

1. 内容类型：面试题 / 笔试题 / LeetCode
2. 专题：如调度策略、资源隔离、故障定位
3. 能力维度：基础知识、系统设计、排障、编码、沟通表达
4. 难度：初级 / 中级 / 高级 / 专家
5. 题型：选择、问答、系统设计、实操分析、编码
6. 建议时长：5 分钟 / 15 分钟 / 45 分钟 / 90 分钟
7. 使用场景：一面 / 二面 / 深挖 / 笔试 / 日常训练
8. 评分标准：关键点命中、推理过程、工程取舍、代码质量

### 5.4 各内容类型建议占比

### 面试题

- 占比建议 45%
- 重点覆盖基础知识、系统设计、故障排查、调度策略

### 笔试题

- 占比建议 25%
- 重点覆盖综合知识核验、场景分析、短答与简答

### LeetCode 练习

- 占比建议 30%
- 重点覆盖调度、优先队列、图搜索、并发控制、资源分配

## 6. 模拟面试记录与面评结构

### 6.1 面试记录字段

1. 候选人信息
2. 目标岗位与级别
3. 面试官信息
4. 场次时间与时长
5. 题目列表与逐题记录
6. 追问内容
7. 候选人关键回答摘要
8. 风险点与亮点
9. 最终结论

### 6.2 面评维度

1. AI Infra 基础认知
2. 调度系统设计能力
3. 故障定位与排障能力
4. 工程实现与落地经验
5. 编码能力
6. 沟通表达与结构化思考
7. 综合录用建议

### 6.3 面评输出格式

- 总评：一句话结论
- 维度评分：1 至 5 分
- 证据摘要：对应题目与回答要点
- 风险标签：知识盲区、经验不足、设计薄弱、表达问题、代码不稳
- 后续建议：进入下一轮 / 补充考察 / 暂缓 / 淘汰

## 7. 问题回流闭环方案

### 7.1 回流触发点

1. 面试官在面评中标记高频错误。
2. 笔试批改中发现普遍误区。
3. 编码题提交中发现重复失败模式。
4. 招聘负责人在复盘会中标记题目区分度不足。

### 7.2 回流对象

1. 题目本身需要修订
2. 参考答案与评分标准需要补全
3. 需要新增专题题单
4. 需要新增训练营式学习路径

### 7.3 闭环流程

1. 记录层：面试、笔试、练习行为全部沉淀。
2. 归因层：问题按知识、设计、工程、编码、表达分类。
3. 治理层：内容维护者定期处理问题池。
4. 发布层：修订内容进入题库和模板。
5. 复盘层：查看题目通过率、面评分布与问题趋势。

## 9. 对研发和架构的落地提示

1. 题目、场次、记录、面评、问题池建议独立建模。
2. 模拟面试与笔试需要共享题库，但评分策略不同。
3. LeetCode 模块需要与题库共享标签与候选人档案能力。
4. 所有行为事件需要预留分析字段，便于后续大数据看板接入。
