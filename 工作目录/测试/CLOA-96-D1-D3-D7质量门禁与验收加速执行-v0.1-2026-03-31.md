## [自动化与环境计划]

1. 自动化计划
   - PR 门禁（<=10 分钟）：API 契约校验 + 核心单测 + 冒烟。
   - 日构建：接口回归 + 集成回归 + 可观测检查。
   - 里程碑构建（D3/D7）：性能基线 + 稳定性演练 + 全量回归。
2. 环境计划
   - Env-A（开发联调）：本地/Kind，快速冒烟与契约比对。
   - Env-B（集成验证）：K8s + Operator + Spark/Flink/Volcano 联调。
   - Env-C（准发布）：仿生产配置，执行 D7 最终 Go/No-Go。
3. 数据计划
   - 固定数据集：正常/边界/异常三类样例。
   - 每次执行必须记录 `requestId` / `traceId`。
   - 缺陷复现数据与脚本统一沉淀到 `evidence/`。
4. 协同计划（后端/前端/云原生）
   - 每日 10:00：阻塞同步会（依赖项状态与风险更新）。
   - 每日 18:00：里程碑日结（测试进度、缺陷、门禁状态）。
   - 统一台账：用例、缺陷、证据三向可追溯。
5. 可复用模板
   - 里程碑证据模板：`工作目录/测试/templates/CLOA-96-里程碑验收证据模板.md`

## [质量门禁与验收标准]

1. 缺陷分级与处置时限
   - P0：2 小时响应，24 小时内修复/回滚方案；未处置则 No-Go。
   - P1：4 小时响应，48 小时内修复或签署豁免。
   - P2：纳入迭代，不阻塞里程碑。
   - P3：跟随版本优化。
2. 性能与稳定性阈值（D7）
   - `GET /platform/overview` P95 < 180ms。
   - `GET /platform/runs` P95 < 280ms。
   - API Pod CPU < 1.4 core/pod。
   - 长稳 4h 无 P0/P1 级中断。
3. 发布门禁（D7）
   - 功能/接口/集成主链路通过率 100%。
   - 无未关闭 P0/P1。
   - 告警链路演练通过，回滚预案可执行。
   - QA 与 CTO 双签完成。
4. 阻塞升级路径
   - 触发条件：出现 P0，或连续两次里程碑未达标。
   - 路径：QA Lead -> CTO -> CEO/Board。
   - 动作：任务转 `blocked`，明确阻塞人、恢复条件、预计恢复时间。
