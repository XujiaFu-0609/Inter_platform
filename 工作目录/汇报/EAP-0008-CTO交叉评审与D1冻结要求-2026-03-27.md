# EAP-0008 CTO 交叉评审与 D1 冻结要求

日期：2026-03-27  
责任人：CTO  
对应任务：CLOA-76

## 1. 评审范围

本次交叉评审覆盖以下已交付子任务：

1. `CLOA-77` 后端 Go + Operator 重构设计；
2. `CLOA-80` 前端 API 对齐与平台可视化草图；
3. `CLOA-81` D1/D3/D7 验收清单与质量门禁。

## 2. 评审结论

### 2.1 通过项

1. 后端方向正确：已明确 Go 模块化单体 + Operator 控制面，且中间件路线对齐 PostgreSQL / Redis / Kafka。
2. 前端方向正确：业务前端与 Grafana 的边界清晰，未出现重复建设时序分析能力的倾向。
3. QA 门禁方向正确：D1/D3/D7 验收与 Go/No-Go 口径已经建立，可作为后续里程碑门禁基线。

### 2.2 待冻结项（D1 必须收敛）

1. `platform/*` 聚合接口的字段字典尚未冻结。
2. `engineType/status/severity/slaTier/queueName` 等统一枚举尚未形成权威字典。
3. Grafana 深链字段（`grafanaDashboardUid`、`grafanaPanelId`、时间窗参数）尚未由后端/控制面统一口径。
4. `InterviewPlatform` / `BatchPipeline` / `StreamPipeline` 的状态字段还缺 Cloud Native Architect 正式回稿。
5. Spark/Flink 与 Volcano 的队列模型仍缺 Big Data Architect 正式回稿。

## 3. D1 冻结要求

### 3.1 Backend Engineer

1. 输出 `GET /api/v1/platform/overview`、`GET /api/v1/platform/runs`、`GET /api/v1/platform/runs/{runId}`、`GET /api/v1/platform/alerts` 的字段清单。
2. 冻结以下 DTO 结构：
   - `PlatformOverview`
   - `PlatformRun`
   - `PlatformRunDetail`
   - `PlatformAlert`
3. 在接口层预留 `grafana*` 深链字段，避免前端硬编码链接规则。

### 3.2 Cloud Native Architect

1. 冻结权威枚举字典：
   - `engineType`
   - `runStatus`
   - `alertSeverity`
   - `slaTier`
2. 冻结 CRD 状态字段：
   - `.status.phase`
   - `.status.conditions`
   - `.status.observedGeneration`
   - `.status.lastReconcileTime`
3. 明确哪些字段由 Operator 提供，哪些字段由业务后端聚合提供。

### 3.3 Big Data Architect

1. 冻结 Volcano 队列命名与 SLA 映射。
2. 冻结 Spark/Flink run 维度的最小可观测字段。
3. 明确 Flink 与 Spark 在平台态 API 中的差异字段与共性字段。

### 3.4 QA Lead

1. 按冻结后的 DTO/枚举/CRD 字段更新 D1 验收用例。
2. 将 `用例ID-缺陷ID-证据路径` 绑定到冻结字段，避免联调期间口径漂移。

## 4. 当前风险判定

1. 当前 D1 最大风险不是“没有方案”，而是“方案之间的字段字典未统一”。
2. 如果 2026-03-28 12:00 前未冻结平台态 DTO 与枚举字典，则前端、后端、QA 会产生并行返工。
3. 因此本项风险等级为 **高**，优先级高于视觉草图和次级页面完善。

## 5. CTO 决策

1. `CLOA-77`、`CLOA-80`、`CLOA-81` 作为方向性交付予以接受。
2. 但上述交付不代表 D1 已完成，D1 是否达标仍取决于：
   - `CLOA-78` 回收 Spark/Flink/Volcano 队列与场景映射；
   - `CLOA-79` 回收 CRD/Controller 与运行基线；
   - 新增集成任务完成平台态 DTO 与字典冻结。
3. 若 D1 前未冻结字典与聚合接口，父任务 `CLOA-76` 需维持 `in_progress` 并标记为 `Conditional Go` 候选，不得误判为完成。
