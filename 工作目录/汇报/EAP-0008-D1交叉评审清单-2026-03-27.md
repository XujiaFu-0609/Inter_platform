# EAP-0008 D1 交叉评审清单

日期：2026-03-27  
责任人：CTO  
对应任务：CLOA-76  
适用对象：CLOA-78 / CLOA-79 / CLOA-82

## 1. 目的

本清单用于在 `2026-03-28 12:00` 后快速完成 `CLOA-78`、`CLOA-79`、`CLOA-82` 的交叉评审，避免 D1 冻结阶段因字段口径、状态边界、调度语义不一致而重复返工。

## 2. 评审输入

1. `CLOA-78`：Spark/Flink/Volcano 一期融合方案
2. `CLOA-79`：CRD/Controller 与运行基线
3. `CLOA-82`：平台接口与枚举字典 D1 冻结稿

## 3. 必查项

### 3.1 `CLOA-78` 必查

1. `slaTier` 与 Volcano `Queue/PriorityClass/PodGroup` 的映射是否唯一且可执行。
2. `queueUtilization.*` 的原始指标口径是否完整：
   - `pendingDepth`
   - `runningCount`
   - `capacity`
   - `utilizationRatio`
3. `engineType` 是否严格限定为 `spark|flink`，以及扩展策略是否明确。
4. Kafka 事件流是否与平台态 API 字段可对上。
5. 是否明确禁止 YARN，并且未留下双轨调度歧义。

### 3.2 `CLOA-79` 必查

1. `InterviewPlatform` / `BatchPipeline` / `StreamPipeline` 的 `.spec` / `.status` 边界是否清晰。
2. `controlPlaneHealth` 对应的 `status.conditions` 是否有明确优先级与降级规则。
3. CRD 状态到 API DTO 的映射是否存在唯一来源。
4. Reconcile 异常时 `healthy/degraded/down` 的阈值与滞后窗口是否明确。
5. 是否区分了 Operator 控制态与业务后端事实态，避免状态职责重叠。

### 3.3 `CLOA-82` 必查

1. `platform/*` DTO 是否已经覆盖 D1 必需接口：
   - `GET /api/v1/platform/overview`
   - `GET /api/v1/platform/runs`
   - `GET /api/v1/platform/runs/{runId}`
   - `GET /api/v1/platform/alerts`
2. 统一枚举字典是否与 `CLOA-78` / `CLOA-79` 回稿保持一致：
   - `engineType`
   - `runStatus`
   - `alertSeverity`
   - `slaTier`
   - `queueName`
3. `grafana*` 深链字段是否明确由后端统一构造。
4. 字段来源矩阵是否能回答 CTO 四问：
   - 字段权威来源
   - 后端聚合来源
   - Operator/CRD 来源
   - Spark/Flink/Volcano 原始运行态来源
5. 是否标清了“D1 可执行口径”与“待 v1.1 联动收口”的边界。

## 4. 交叉一致性检查

1. `CLOA-78` 的 `slaTier/queueName` 必须与 `CLOA-82` 的 DTO 和枚举一致。
2. `CLOA-79` 的 `controlPlaneHealth/status.conditions` 必须与 `CLOA-82` 的字段来源矩阵一致。
3. `CLOA-78` 与 `CLOA-79` 不能分别定义不同的健康状态或优先级语义。
4. QA 的 D1 验收项必须能直接映射到以上字段，不允许再出现“描述性口径”。

## 5. 评审结论模板

### Go

适用条件：

1. 三份回稿全部齐备；
2. 字段、状态、队列口径一致；
3. QA 可按冻结稿直接执行 D1 验收。

### Conditional Go

适用条件：

1. `CLOA-82` 已冻结；
2. `CLOA-78` 或 `CLOA-79` 尚有少量未落章项；
3. 但不影响 QA 对 D1 核心边界冻结进行验收。

### No-Go

适用条件：

1. `CLOA-78` 或 `CLOA-79` 仍无正式初稿；
2. 或关键字段语义冲突；
3. 或出现违反技术约束的方案（例如引入 YARN、非 Go 正式后端主线）。

1. `2026-03-28 12:00`：按本清单评审 `CLOA-78/79/82` 最新稿。
